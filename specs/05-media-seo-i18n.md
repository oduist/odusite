# Media, SEO, i18n

## Images

- Source of truth: Odoo's `/web/image/<model>/<id>/<field>/<WxH>?unique=<hash>`
  (public route, on-the-fly resize, placeholder fallback, `unique` checksum →
  1-year immutable caching).
- API responses return `image` fields as such relative paths, with `unique`
  already embedded (computed from the record's `write_date`).
- The Astro site rewrites them to `/img/**` — a Worker route that proxies to
  `ODOO_URL` and caches on the edge (`cf.cacheEverything`, respects `unique`).
  The Odoo host is never exposed to the browser.
- Portal/protected binaries (invoices PDF, attachments) are NOT served via
  `/img` — they are streamed through authenticated `/odusite/v1/...` endpoints.

## SEO

- Every detail entity carries `seo: {title, description, keywords, og_image}`
  mapped from `website_meta_title/description/keywords/og_img`, falling back to
  the entity name/teaser/cover.
- The site renders canonical URLs, OpenGraph/Twitter meta, and JSON-LD:
  `Product` (from combination info), `BlogPosting`, `Event`, `JobPosting`.
- `sitemap.xml` is generated by the site (Astro endpoint): static routes from
  enabled blocks + entity URLs fetched from `GET /odusite/v1/sitemap`
  (odusite_base aggregates slugs + write dates of all published entities from
  installed odusite modules).
- `robots.txt` is served by the site (static, per environment).
- Redirects: `website.rewrite` records are exposed via
  `GET /odusite/v1/redirects` and compiled into the Worker's redirect map.

## i18n

- Available languages come from `GET /odusite/v1/site` (`website.language_ids`,
  `default_lang_id`, url codes).
- URL scheme mirrors Odoo: default language unprefixed, others `/<url_code>/...`.
  The site passes `?lang=<code>` on every API call; Odoo activates the lang
  context so translated fields (translate=True / html_translate) come localized.
- `hreflang` alternates rendered for every localized page.
- UI strings of the site itself live in the site's i18n catalog
  (`site/src/i18n/<lang>.json`) — not in Odoo.
- The `od_lang` cookie stores the visitor's choice; language switcher is part
  of the core layout.
